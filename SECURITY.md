# Security Policy

## Supported versions

This is an active research repository. Security fixes are applied on the latest
`main` state.

## Reporting a vulnerability

Please open a private report if possible. If private reporting is not
available, open a minimal public issue without exploit details and tag it
`security`.

Include:
- affected file/module,
- impact summary,
- reproduction steps,
- suggested mitigation if available.

## Notes

- Do not post secrets, tokens, or private infrastructure details in issues.
- Dependency updates may be accepted quickly when risk is clear.
